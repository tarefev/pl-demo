# Каркас работы чата (детерминированное поведение)

## Типы ввода

1. **Пилз-звёздочка** — пункт из выпадайки-звёздочки (ИИ-меню блока документа); содержит id действия и ссылку на блок.
2. **Чоис** — нажатие кнопки, предложенной чатом; содержит id.
3. **Текст** — свободный текстовый ввод.
4. **Файл** — приложенный документ; может прийти вместе с текстом.

## Глобальные правила (действуют в любом состоянии, кроме D)

- **Сброс.** «стоп», «в меню», «главное меню», «начать сначала», «сброс» → бросить текущий сценарий без подтверждения, выйти в C. Если тип документа ещё не выбран — следующий ввод снова вернёт к выбору типа (A).
- **Отмена.** «отмена», «отмени», «прекрати» в подсценарии (B) → выйти из подсценария в C. В состоянии A не действует (выбор типа обязателен). Если у текущего вопроса есть собственная кнопка «Отмена/Назад» — она в приоритете (см. «текст = кнопка»).
- **Текст = кнопка.** Если введённый текст дословно совпадает с одной из показанных сейчас кнопок — считаем это нажатием кнопки. Исключение: необратимые действия не выполняем по тексту — переспрашиваем: «действие необратимо, если хотите продолжить — нажмите кнопку».
- **Валидность чоиса.** Чоис валиден, только если относится к актуальному вопросу чата (последние показанные кнопки, включая служебные: «Прервать?», триада B.2.3.2, уточняющий вопрос C.3.1.2). Протухший чоис (кнопка из старого сообщения в истории) → отбивка «действие недоступно» + повтор актуального вопроса, если он есть.
- **Подтверждение правок документа.** Если есть ожидающие подтверждения правки (pending patch), короткий текст «да / ок / применить / +» = применить, «нет / отмена / не надо» = отменить.

## Служебный вопрос «Прервать?»

Там, где матрица требует прерывания, чат спрашивает: «Прервать текущий сценарий и перейти к новому запросу? Введённые данные и наработки не сохранятся.» + кнопки **«Прервать»** / **«Продолжить текущий»**.

- «Прервать» → текущий сценарий сбрасывается, исходный ввод (который вызвал вопрос) выполняется по правилам состояния C.
- «Продолжить текущий» → сценарий продолжается с того же места (повторяет свой вопрос/кнопки).
- Любой другой ввод во время этого вопроса (текст, файл, пилз) = «Прервать» + выполнить этот новый ввод. `[открытый вопрос — подтвердить]`

## A. Сценарий выбора типа документа

Запускается принудительно: любой ход по документу без выбранного типа (в т.ч. авто-приветствие при первом открытии). Это единственный обязательный шаг перед свободным вводом.

**Завершение сценария:**
- Текст — «Тип документа выбран».
- Побочные эффекты: заголовок документа (H1) по выбранному типу, переименование «Нового документа».
- Выход в состояние C.

Реакции на ввод:

- **A.1 пилз-звёздочка** → игнорируем, снова показываем чоисы выбора типа с поясняющим текстом.
- **A.2 чоис** → если это чоис пикера типа — продолжаем сценарий; чужой/протухший id → игнорируем + reprompt.
- **A.3 текст** (определитель):
  - A.3.1 текст совпадает с кнопкой пикера → выбираем её;
  - A.3.2 текст распознан как название типа документа (в т.ч. реальный документ вне списка → тип «другое» + сохранение конкретного названия) → устанавливаем тип;
  - A.3.3 вольный ввод (не тип документа) → игнорируем, снова чоисы с поясняющим текстом.
- **A.4 файл** → игнорируем (вложение отбрасывается), снова чоисы с поясняющим текстом.

## B. Работает подсценарий

### B.1 Ждёт выбор (показаны чоисы)

- **B.1.1 пилз-звёздочка** → «Прервать?»:
  - прервать → выполнить пилз согласно id;
  - не прерывать → снова чоисы текущего шага с поясняющим текстом.
- **B.1.2 чоис** → продолжение сценария. Несколько чоисов сразу на шаге мульти-выбора = подтверждение выбора.
- **B.1.3 текст** (определитель):
  - B.1.3.1 текст совпал с кнопкой → нажать её (необратимые — через переспрос, см. глобальные правила);
  - B.1.3.2 вольный ввод → «Прервать?»:
    - прервать → обработать ввод по правилам состояния C (C.3: запуск сценария / уточнение / ответ);
    - не прерывать → снова чоисы с поясняющим текстом.
- **B.1.4 файл** → «Прервать?»:
  - прервать → подсценарий «Разбор файла» (как C.4);
  - не прерывать → снова чоисы с поясняющим текстом.

### B.2 Ждёт свободный ввод

По умолчанию свободный ввод — текст. Отдельный вопрос сценария может явно допускать документ как ответ (вариант свободного ввода); сейчас таких вопросов нет, ветка зарезервирована.

- **B.2.1 пилз-звёздочка** → «Прервать?»:
  - прервать → выполнить пилз согласно id;
  - не прерывать → снова задать текущий вопрос, ждать свободный ввод.
- **B.2.2 чоис** → валидны только кнопки актуального служебного вопроса (триада, «Прервать?» — см. глобальные правила); иначе — отбивка «действие недоступно» + повтор вопроса.
- **B.2.3 текст** (определитель):
  - B.2.3.1 текст подходит под контекст вопроса → принимаем как ответ, продолжаем сценарий;
  - B.2.3.2 текст не подходит → триада: «Это было ответом, нужно переформулировать вопрос, или это новый вопрос?» — 3 чоиса:
    - «Это был ответ» → принять как ответ;
    - «Переформулировать вопрос» → снова задать текущий вопрос, ждать ввод;
    - «Это новый вопрос» → выйти из сценария без дополнительного подтверждения, обработать ввод по правилам C.
- **B.2.4 файл**:
  - если текущий вопрос сценария явно допускает документ как ответ → принять документ как ответ (через определитель контекста, аналогично B.2.3);
  - иначе, если вместе с файлом есть текст → файл игнорируем с отбивкой («документ пропущен»), текст обрабатываем как B.2.3;
  - иначе (только файл) → отбивка «жду текстовый ответ на текущий вопрос», продолжаем ждать.

## C. Ни один сценарий не запущен, тип документа выбран (свободный ввод)

- **C.1 пилз-звёздочка** → выполнить согласно id (запуск соответствующего сценария / действия над блоком).
- **C.2 чоис** → валидны кнопки актуального вопроса (например уточняющего C.3.1.2); протухшие → отбивка «действие недоступно».
- **C.3 текст** (определитель):
  - C.3.1 найден подсценарий:
    - C.3.1.1 уверенность высокая → запускаем сценарий;
    - C.3.1.2 уверенность средняя → уточняющий вопрос с пилзами-кандидатами + вариант «Ничего из этого»:
      - пилз выбран → запускаем сценарий;
      - отказ («Ничего из этого») → текст «запуск не выполняем, уточните, что нужно сделать»; остаёмся в C;
  - C.3.2 действие над документом (правка блока/фрагмента; требует doc_ref) → подсценарий «Действие над документом»: предложить правку, применить только после подтверждения (кнопки «Применить/Отменить» или «да/нет» — см. глобальные правила);
  - C.3.3 иначе (сценарий не найден / уверенность низкая) → ответить на свободный ввод генерацией текста согласно контексту.
- **C.4 файл** → запускаем подсценарий «Разбор файла» (определение типа документа — внутри разбора).

---

## D. Идёт разбор документа

Любой ввод (1–4) → отбивка «идёт разбор документа, действия недоступны». Ввод разблокируется по завершении разбора (итог разбора чат присылает сам).

**Отдельно (не состояние, а занятость хода):** пока чат думает / выполняет tools / генерирует ответ — новый ввод не принимается до конца текущего хода (фронт блокирует ввод, бэк отвечает «занято»).

## Открытые вопросы

1. Ввод «не кнопкой» во время вопроса «Прервать?» — сейчас зафиксировано как «прервать + выполнить новый ввод». Подтвердить или сменить на повтор вопроса.
2. Полосы уверенности в C.3.1: зафиксировано «высокая → запуск, средняя → уточнение, низкая → ответ текстом» (в исходнике средняя и низкая были склеены в уточнение). Подтвердить.
3. Текст завершения A («Тип документа выбран») — дословный контракт или примерный смысл.

## Известные расхождения с кодом (код не правим, фиксируем)

- Полная сверка спека↔код — отдельный шаг (после утверждения этого файла).
- Пилз-звёздочка в коде распознаётся по тексту пиллы (id выводится из текста через реестр блок-меню), а не по прямому id из фронта.
- Ветка B.2.4 «вопрос допускает документ» в коде не реализована (таких вопросов нет) — зарезервировано.
